# GSC Phase 2 Indexation Audit

This document provides a comprehensive technical audit of the remaining Google Search Console (GSC) indexation issues for `mevaclinic.com`. Every URL was verified live against the production site in June 2026.

## Executive Summary

- **Total audited URLs:** 48 URLs (18 in Group A, 16 in Group B, 4 in Group C, 1 in Group D, 9 in Group E).
- **Group A (Page with redirect):** 100% expected. All source URLs redirect to correct canonical targets. None are present in the sitemap.
- **Group B (Alternate canonical / duplicate-like):** 15/16 URLs behave correctly (non-www redirects to www, packages redirect to treatments). **1 URL returning 404 requires fixing:** `/en/piezo-rhinoplasty`.
- **Group C (Possible noindex / wrong hair transplant slug):** 100% expected. All 4 localized hair-transplant slugs redirect correctly to `/treatments/meva-mixed-hair` which is 200 OK and indexable.
- **Group D (Definite route generation bug):** `/ro/undefined` returns 404. It is not generated by the codebase or sitemap. **Requires a safe fallback redirect.**
- **Group E (Crawled, currently not indexed):** 8/9 are high-quality, technically perfect pages (200 OK, in sitemap, self-referencing canonical, no noindex). Favicon query URL is a non-page. No technical issues found; indexation is pending crawler recrawl.

#### Root Cause Analysis of `/ro/undefined`
We ran code scans for occurrences of `undefined` and verified all dynamic data lists (treatmentsData and blogPosts) for missing slugs or IDs. All entries have valid, non-null, non-undefined slugs. The sitemap generation is also correct.
The `/ro/undefined` crawl was likely triggered by a legacy template bug (since resolved) where `undefined` was generated on client-side routing. We will implement a catch-all route-level redirect in `next.config.ts` from `/:lang/undefined` to `/:lang/about-us` (or `/ro/undefined` specifically to `/ro/about-us`) to satisfy Google Search Console crawlers.

## Recommended Action Plan

### 1. Fix `/en/piezo-rhinoplasty` 404
Modify the redirect rule in `next.config.ts` from:
```ts
{
  source: '/ro/piezo-rhinoplasty',
  destination: '/ro/treatments/piezo-rhinoplasty',
  permanent: true,
}
```
to a multilingual pattern:
```ts
{
  source: '/:lang(en|de|ro|es|it|ru|fr)/piezo-rhinoplasty',
  destination: '/:lang/treatments/piezo-rhinoplasty',
  permanent: true,
}
```

### 2. Catch and Redirect `/undefined` crawl bugs
Add the following redirect in `next.config.ts` to cleanly handle any legacy crawler requests for `/undefined` paths:
```ts
{
  source: '/:lang(en|de|ro|es|it|ru|fr)/undefined',
  destination: '/:lang/about-us',
  permanent: true,
}
```
This maps `/ro/undefined` to `/ro/about-us` and general language undefined paths to their respective About Us landing pages, resolving the 404 status.
